Agreement Engine

Here is a list of every command that can be used with Agreement Engine. For a full explanation of the system and how it works, see the about page. In order to run a command, send a tweet starting with @agreementengine, followed by the command and its arguments.

generate (number) (likes/retweets)

Generates new like or retweet contracts, awarding the user with TSC. If no parameters are given, it defaults to generating 10 like contracts.
ex: @agreementengine generate 5 retweets
try it yourself!

execute (number) | in reply to

Spends TSC to execute other people's contracts on the post replied to. This command must be executed in response to another post.
ex: @agreementengine execute 100

balance

Gives the user's current balance in TSC.
ex: @agreementengine balance
try it yourself!

likes

Displays how many active like contracts the user has. A user can have a maximum of 10 active like contracts at a time.
ex: @agreementengine likes
try it yourself!

retweets

Displays how many active retweet contracts the user has. A user can have a maximum of 10 active retweet contracts at a time.
ex: @agreementengine retweets
try it yourself!

agreement (number) (type) | tag other users

Creates a new agreement between the user and the first other user mentioned in the post. If no parameters are given, an unenforced contract is created. The creator of the agreement can stake TSC or like/retweet contracts which are exchanged for TSC if the agreement is broken. If the agreement is upheld, the creator keeps their stake, otherwise it is given to the other user.
ex: @agreementengine agreement 500 @otheruser I promise to visit next time I'm in LA!
ex: @agreementengine agreement 5 likes @otheruser I'll pay you back for dinner last night.
try it yourself!

upheld | in reply to

Votes that an agreement was upheld. Must be a response to an agreement post.
ex: @agreementengine upheld

broken | in reply to

Votes that an agreement was broken. Must be a response to an agreement post.
ex: @agreementengine broken
